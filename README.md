# YemenBiz Ledger AI

**YemenBiz Ledger AI** هو مشروع Python مفتوح المصدر لبناء دفتر تجاري ذكي للمحلات الصغيرة في اليمن.

الأداة تساعد صاحب المحل على إدارة:

- المبيعات اليومية.
- الديون الآجلة على العملاء.
- العملاء.
- المنتجات.
- المخزون.
- التقارير اليومية.
- إدخال المبيعات باللغة العربية الطبيعية باستخدام الذكاء الاصطناعي أو محلل داخلي بدون API.

---

## 1. لماذا هذا المشروع؟

كثير من المحلات الصغيرة في اليمن تعتمد على الدفاتر الورقية أو رسائل واتساب لتسجيل المبيعات والديون. هذا يسبب مشاكل عملية:

- ضياع الديون.
- عدم معرفة الربح الحقيقي.
- ضعف متابعة المخزون.
- صعوبة معرفة العملاء المتأخرين في السداد.
- صعوبة معرفة المنتجات الأكثر مبيعًا.
- عدم وجود تقارير يومية واضحة.
- عدم وجود فاتورة أو سجل مرتب للعمليات.

هذا المشروع يحاول حل هذه المشكلة بأداة بسيطة، عربية، قابلة للتطوير، ومناسبة للسوق المحلي.

---

## 2. ما الذي يجعل الأداة مفيدة تجاريًا؟

الأداة ليست مجرد مشروع تدريبي. يمكن تطويرها لاحقًا إلى منتج مدفوع يخدم:

- البقالات.
- محلات الجملة.
- الصيدليات الصغيرة.
- محلات الملابس.
- محلات الجوالات.
- المطاعم الصغيرة.
- محلات مواد البناء.
- الموردين المحليين.

صاحب المحل قد يدفع مقابل أداة تساعده على معرفة أمواله وديونه ومخزونه بشكل واضح.

---

## 3. المزايا الحالية في نسخة MVP

### 3.1 إدارة العملاء

- إضافة عميل.
- حفظ رقم الهاتف والعنوان والملاحظات.
- ربط المبيعات الآجلة باسم العميل.

### 3.2 إدارة المنتجات

- إضافة منتج.
- حفظ سعر الشراء.
- حفظ سعر البيع.
- حفظ كمية المخزون.
- تحديد الوحدة: كرتون، كيس، علبة، قطعة، كيلو.

### 3.3 إدارة المبيعات

- تسجيل عملية بيع.
- دعم الدفع النقدي.
- دعم الدفع الآجل.
- دعم التحويل.
- حساب الإجمالي.
- حساب المدفوع.
- حساب المتبقي كدين.

### 3.4 إدخال بيع باللغة العربية

مثال:

```text
بعت لأحمد 3 كرتون ماء و2 كيس رز آجل بـ 18500 ريال
```

تقوم الأداة بتحويل النص إلى بيانات منظمة:

```json
{
  "customer_name": "أحمد",
  "payment_type": "credit",
  "total_amount": 18500,
  "paid_amount": 0,
  "remaining_amount": 18500,
  "items": [
    {
      "product_name": "كرتون ماء",
      "quantity": 3
    },
    {
      "product_name": "كيس رز",
      "quantity": 2
    }
  ]
}
```

### 3.5 لوحة تحكم

تعرض:

- إجمالي المبيعات.
- إجمالي المبالغ المحصلة.
- إجمالي الديون.
- عدد العملاء.
- عدد المنتجات.
- المنتجات منخفضة المخزون.
- أكثر المنتجات مبيعًا.

### 3.6 تقرير عربي مختصر

تولد الأداة تقريرًا عربيًا بسيطًا يمكن لصاحب المحل فهمه بسرعة.

## 4. التقنية المستخدمة

المشروع مبني بالكامل بـ Python:

| التقنية | الاستخدام |
|---|---|
| FastAPI | بناء Backend API |
| SQLite | قاعدة بيانات محلية للنسخة الأولى |
| SQLAlchemy | التعامل مع قاعدة البيانات |
| Pydantic | التحقق من البيانات |
| Streamlit | واجهة مستخدم سريعة |
| OpenAI | اختياري لتحليل النصوص |
| pytest | اختبارات |
| ruff | فحص جودة الكود |

## 6. طريقة التشغيل لاحقًا

### 6.1 إنشاء البيئة

```powershell
python -m venv .venv
.\.venv\Scripts\activate
```

### 6.2 تثبيت المكتبات

```powershell
pip install -r backend\requirements.txt
```

### 6.3 تجهيز ملف البيئة

```powershell
Copy-Item .env.example .env
```

### 6.4 تشغيل Backend

```powershell
uvicorn backend.app.main:app --reload
```

افتح التوثيق التفاعلي:

```text
http://127.0.0.1:8000/docs
```

### 6.5 تشغيل الواجهة

افتح Terminal آخر:

```powershell
streamlit run frontend\streamlit_app.py
```

## 7. ملاحظات مهمة

- الأداة تعمل بدون OpenAI API لأنها تحتوي على محلل داخلي Rule-Based.
- OpenAI اختياري ويمكن تفعيله لاحقًا من ملف `.env`.
- قاعدة البيانات الافتراضية SQLite ومناسبة للتجربة.
- عند التحويل إلى منتج حقيقي، يفضل استخدام PostgreSQL وإضافة نظام مستخدمين وصلاحيات.

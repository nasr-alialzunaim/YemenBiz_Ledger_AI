# المعمارية الفنية لمشروع YemenBiz Ledger AI

هذا الملف يشرح البنية الفنية للمشروع وكيف تتفاعل مكوناته.

---

## 1. نظرة عامة

المشروع مكوّن من قسمين رئيسيين:

1. **Backend API** مبني باستخدام FastAPI.
2. **Frontend MVP** مبني باستخدام Streamlit.

قاعدة البيانات المستخدمة في النسخة الأولى هي SQLite، والهدف منها تسهيل التشغيل المحلي والتجربة السريعة.

---

## 2. طبقات المشروع

### 2.1 طبقة API

المسار:

```text
backend/app/api/
```

هذه الطبقة تحتوي على Routes التي يستدعيها المستخدم أو الواجهة.

أمثلة:

- `customers_routes.py`
- `products_routes.py`
- `sales_routes.py`
- `ai_routes.py`
- `reports_routes.py`

وظيفة هذه الطبقة:

- استقبال الطلبات HTTP.
- التحقق الأولي من البيانات.
- استدعاء خدمات Business Logic.
- إرجاع Response مناسب.

---

### 2.2 طبقة Schemas

المسار:

```text
backend/app/schemas/
```

تحتوي على نماذج Pydantic.

وظيفتها:

- تحديد شكل Request.
- تحديد شكل Response.
- التحقق من نوع البيانات.
- توثيق API تلقائيًا داخل Swagger.

---

### 2.3 طبقة Services

المسار:

```text
backend/app/services/
```

هذه أهم طبقة لأنها تحتوي منطق التطبيق.

أمثلة:

- `ai_parser.py`: تحليل البيع المكتوب باللغة العربية.
- `sales_service.py`: إنشاء المبيعات وتحديث المخزون.
- `product_service.py`: إدارة المنتجات.
- `customer_service.py`: إدارة العملاء.
- `report_service.py`: التقارير ولوحة التحكم.

---

### 2.4 طبقة Database

المسار:

```text
backend/app/db/
```

تحتوي على:

- `database.py`: الاتصال بقاعدة البيانات.
- `models.py`: جداول SQLAlchemy.

## 3. تدفق عملية بيع عادية

```text
المستخدم يرسل طلب بيع
↓
sales_routes.py
↓
sales_service.py
↓
حساب الإجمالي والمتبقي
↓
تحديث المخزون
↓
حفظ Sale و SaleItem
↓
إرجاع نتيجة للمستخدم
```

---

## 4. تدفق عملية بيع باللغة العربية

```text
المستخدم يكتب:

↓
ai_routes.py
↓
NaturalLanguageSaleParser
↓
تحليل العميل، المنتجات، المبلغ، نوع الدفع
↓
إرجاع JSON منظم
↓
اختياريًا حفظه كعملية بيع
```

---

## 5. لماذا استخدمنا Streamlit؟

Streamlit مناسب للـ MVP لأنه يسمح ببناء واجهة بسرعة بدون الحاجة إلى React أو Flutter في البداية.

الهدف من الواجهة الحالية:

- تجربة المنتج.
- عرض الفكرة.
- اختبار احتياج السوق.
- تقديم Demo لصاحب محل أو مستثمر أو جهة داعمة.

---

## 6. كيف يمكن تطوير المعمارية لاحقًا؟

عند تحويل المشروع إلى منتج حقيقي:

- استبدال SQLite بـ PostgreSQL.
- إضافة Authentication.
- إضافة صلاحيات: مدير، كاشير، محاسب.
- فصل Frontend باستخدام React أو Flutter.
- إضافة Docker.
- إضافة Background Jobs.
- إضافة نسخ احتياطي تلقائي.
- إضافة تقارير PDF.
